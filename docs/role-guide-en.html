1. What is Role & User Management?

The Role & User Management feature allows you to control who can access your system and what they can do.

Roles

Roles are collections of permissions that you can assign to users. Examples:

  • Cashier - Can only process sales and view inventory
  • Manager - Can view reports and manage inventory
  • Owner - Has full access to all features
  • Delivery Driver - Can only access delivery list
📝 Important Note: You can create roles with any name that suits your business needs. This system is highly flexible and can be customized to match your organizational structure.

Users

Users are people who can log into the system. Each user can have one or more roles that determine what they can access.

Super Admin

A special role named "Super Admin" has unrestricted access to all features. This role doesn't require additional permissions.

⚠️ Warning: Only grant Super Admin role to highly trusted individuals. This role has complete unrestricted access!

2. Accessing Role & User Menu

Access Requirements

To access the Role & User menu, you need special permission:

  • Manage Access Control - Permission to manage roles and users
  • Or have Super Admin role

How to Access:

  1. Open the application
  2. Tap "Functions" menu
  3. Scroll down to "Access Control" section
  4. Select "Role" or "User"
💡 Don't see this menu? Contact your administrator or business owner to get the required access permissions.

3. Managing Roles

Viewing Role List

After opening the Role menu, you'll see all created roles along with the number of users in each role.

Creating a New Role

  1. Open the Role page
  2. Tap "Add Role" button or + icon
  3. Enter Role Name (e.g., "Cashier", "Manager", "Driver")
  4. Select Permissions to grant this role (see Permissions section below)
  5. Tap "Save"

Editing a Role

  1. Tap on the role you want to edit
  2. Modify the name or permissions as needed
  3. Tap "Save Changes"
💡 Tip: Permission changes will immediately apply to all users with that role.

Deleting a Role

  1. Tap on the role you want to delete
  2. Tap "Delete Role" button
  3. Confirm the deletion
⚠️ Caution: Make sure no users are still assigned to this role. If there are, move them to another role first.

4. Managing Users

Viewing User List

On the User page, you can see all registered users along with their roles.

Adding a New User

  1. Open the User page
  2. Tap "Add User" button or + icon
  3. Fill in user information:
    • Full Name
    • Email (for login)
    • Password (initial password)
    • Phone Number (optional)
  4. Select Role(s) for this user (can be multiple)
  5. Tap "Save"
📝 Note: Users can have multiple roles. All permissions from all roles will be combined.

Editing a User

  1. Tap on the user you want to edit
  2. Modify information or roles as needed
  3. To change password, enable "Change Password" option
  4. Tap "Save Changes"

Deactivating a User

  1. Tap on the user you want to deactivate
  2. Tap "Deactivate User" button
  3. Confirm the action
💡 Tip: Deactivating is better than deleting if you might need to reactivate the user in the future.

6. Notifications & Devices

The system supports push notifications to keep your team informed about important activities.

Available Notification Types

1. Sales Notifications

Permission: Receive Sales Notification

Users with this permission will receive notifications about:

  • New sale completed
  • Payment added (for debt sales)
  • Debt payment completed
  • Sale voided
  • Sale returned
  • Sale refunded
  • Delivery confirmed

2. Inventory Notifications

Permission: Receive Inventory Notification

Users with this permission will receive notifications about:

  • Products out of stock
  • Products below reorder threshold (low stock warning)
📝 Note: Inventory notifications are sent automatically after sales transactions that cause stock changes.
📱 Master Notification Permission: To receive any notifications, users must have the Receive Notification permission plus the specific permission (Sales or Inventory).

Managing Notification Devices

Users with Manage Push Notification Devices permission can:

  • View all devices registered for notifications
  • Remove inactive devices
  • Test notification delivery

How to Enable Notifications

  1. Ensure user has appropriate notification permissions
  2. Login to the app on device
  3. Allow notifications when prompted by the system
  4. Device will be automatically registered
⚠️ Note: Notifications only work on devices that are logged in and have granted notification permission in device settings.

7. Tips & Tricks

🎯 Best Practices

Start with Templates Create basic roles like "Cashier", "Manager", "Owner" first
Use Clear Names Give roles names that are easy to understand (e.g., "Morning Shift Cashier" vs "Role1")
Regular Review Check each role's permissions at least once a month
Documentation Record who has access to what and why
Strong Passwords Use combination of uppercase, lowercase, numbers, and symbols

🔒 Security

  • Don't share accounts - Each person should have their own account
  • Change default passwords - Ask new users to change password after first login
  • Deactivate immediately - Deactivate accounts of employees who have left
  • Monitor activity - Check activity logs regularly
  • Limit Super Admins - Maximum 2-3 people only

⚡ Efficiency

  • Position-based roles - Create roles for each position in your team
  • Use multiple roles - Grant multiple roles if needed rather than creating complex new roles
  • Template for new roles - Duplicate existing roles when creating similar ones

Small Store (2-5 people)

8. Troubleshooting

❌ User Cannot Login
Problem: User cannot access the system

Solution:
  • Check if account is active (not deactivated)
  • Ensure email and password are correct
  • Reset user password if forgotten
  • Check internet connection
  • Try logging out and back in
🚫 Menu Not Visible
Problem: User doesn't see certain menus

Solution:
  • Check user's role - ensure they have appropriate permissions
  • Verify permissions in that role are correct
  • Ask user to logout and login again
  • Check if user is in the correct outlet
🔔 Notifications Not Received
Problem: User not receiving notifications

Solution:
  1. Ensure user has "Receive Notification" permission
  2. Check specific permission (Sales or Inventory) is granted
  3. Verify notification permission in device settings
  4. Ensure app is not in battery saver mode
  5. Try logging out and back in
  6. Reinstall app if still problematic
⚠️ Cannot Delete Role
Problem: Role cannot be deleted

Solution:
  • Check if there are still users with this role
  • Move all users to another role first
  • Once no users remain, role can be deleted
🔄 Permission Changes Not Applied
Problem: After changing permissions, user still can't access

Solution:
  • Ask user to logout and login again
  • Wait a moment for data synchronization
  • Double-check permissions were saved correctly
  • Ensure no other role is restricting access

❓ Frequently Asked Questions

Q: What's the maximum number of roles I can create?

A: There's no limit to the number of roles. You can create as many as needed for your business requirements.

Q: Can a user have more than one role?

A: Yes! A single user can have multiple roles. All permissions from all roles will be combined and the user will have access to all features allowed by those roles.

Q: What's the difference between "Super Admin" and a role with all permissions?

A: Super Admin is a special role that automatically has all permissions without needing configuration. This role will also automatically get access to new features added in the future. Regular roles with all permissions must be manually configured for each permission.

Q: What if I forget a user's password?

A: You can reset the user's password by editing that user and enabling the "Change Password" option, then entering a new password.

Q: Can deactivated users still login?

A: No. Deactivated users cannot login to the system. You can reactivate them anytime if needed.

Q: Can I see user activity?

A: Yes, the system records all transactions and data changes along with information about who performed them. You can see this in reports or the history of each data entry.

Q: How do I create a role that can only view but not modify data?

A: Grant only "View" permissions without "Manage" permissions. Example: grant "View Inventory" without "Manage Inventory" for users who should only view stock.

Q: Do notifications work on all devices?

A: Notifications will be sent to all devices where the user is logged in and has granted notification permission. One user can receive notifications on multiple devices simultaneously.

Q: What's the best way to set permissions for cashiers?

A: For cashiers, grant at minimum: "Process Sales", "View Inventory", and "Receive Sales Related Notification". Avoid granting permission to modify sales history or manage prices for better security.
